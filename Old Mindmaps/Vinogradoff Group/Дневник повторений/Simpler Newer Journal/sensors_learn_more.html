About Sensors

What is a Sensor?

A sensor is any IoT device. In other words, any electronic device that is able to be triggered because of external events, and when that happens it's able to send URI requests to this website.

What types of Sensors exist?

As long as it can send URI requests, implementation is up to you. It can be a piece of software, it can be an Arduino-based device, it can be a smartphone app, it can even be a camera with AI engine that is set up to send certain repetitions to your account once it sees some kind of object in front of it.

How to set up a Sensor?

In your account, click "Add a new Sensor" button. It will generate API Key & Secret. Then get a device (E.g. Arduino), and tell it to send a JSON API request to our website every time it's triggered. An example of such code would be:

def activate
  if triggered?
    send_request(json_data);
  end
end

def json_data
 {
    API_Key: "9s8v7f9s8f7vsss";
    API_Secret: "ksjfvnksdfnveknk34n3kj3n";
    event_name: "My Door Sensor Was Triggered";
    event_description: "Someone opened a door";
 }
end

def send_request(data)
  %x(curl -d "#{data}")
end

What can be a trigger?

In Arduino board there is an entire world of sensors available (photo sensors, motion sensors, buttons, chemical sensors etc). Most of them are Plug & Play. Or if you use a camera capable of AI object recognition, you add a code to it like so:

def ai_recognized_an_object
   object_name = AILibrary.recognize
   activate(object_name)
end

def activate(object_name)
  if triggered?
    send_request(json_data(object_name));
  end
end

def json_data(name)
 {
    API_Key: "9s8v7f9s8f7vsss";
    API_Secret: "ksjfvnksdfnveknk34n3kj3n";
    event_name: "Sensor saw a #{name}";
    event_description: "#{name} appeared in front of a camera";
 }
end

def send_request(data)
  %x(curl -d "#{data}")
end
